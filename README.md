# fnb-trend-lifecycle
## F&B 트렌드 생애주기 빅데이터 분석 파이프라인

> 왜 어떤 F&B 유행은 K-문화로 진화하고, 다른 유행은 사회적 피로와 함께 사라지는가?


## 문제 정의

최근 F&B 트렌드는 K-콘텐츠(숏폼 등)를 통해 빠르게 확산되지만,
서로 다른 생애주기와 사회적 영향을 보인다.

- **K-문화 기회형** (예: 불닭) — 긍정적 콘텐츠로 진화, 장기적 브랜드로 성장
- **단기 휘발형** (예: 탕후루) — 급격한 침체, 상권 붕괴, 사회적 피로 야기

이 두 유형을 가르는 데이터적 패턴을 빅데이터 파이프라인으로 분석한다.

### 분석 질문
1. 유행의 초기 확산 속도와 지속성 사이에 패턴이 있는가?
2. 부정 여론 비율이 높을수록 쇠퇴 속도가 빠른가?
3. K-문화형과 휘발형을 가르는 결정적 변수는 무엇인가?


## 기술 스택

| 단계 | 도구 |
|------|------|
| 데이터 수집 | Python (pytrends, BeautifulSoup, YouTube Data API) |
| 저장 | HDFS |
| 처리/전처리 | Apache Spark |
| 분석 | Apache Hive (HiveQL) |
| 시각화 | Matplotlib / Plotly |


## 데이터 소스

| 소스 | 수집 방법 | 데이터 |
|------|----------|--------|
| Google Trends | pytrends | 검색량 시계열 |
| 구글 뉴스 RSS | RSS 파싱 | 기사 제목 |
| 네이버 뉴스 | BeautifulSoup | 기사 본문 |
| 네이버 카페 | BeautifulSoup | 커뮤니티 반응 |
| 유튜브 댓글 | YouTube Data API | 영상 반응 |


## 파이프라인 (구현 계획)

```
[수집]
pytrends           → 키워드별 검색량 시계열 데이터 (Google Trends)
구글 뉴스 RSS      → 기사 제목
네이버 뉴스/카페   → 기사 본문 / 커뮤니티 반응 텍스트
유튜브 댓글        → 영상 반응 텍스트

[저장]
HDFS 적재 (CSV, JSON 포맷)

[전처리 - Spark]
결측치 처리 / 텍스트 정제 / 감성 점수 산출

[분석 - Hive]
키워드별 검색량 추이 집계
유형별 부정 여론 비율 비교

[분석 - Spark MLlib]
검색량 패턴 및 감성 데이터 기반 유행 유형 분석
(군집화 또는 시계열 패턴 비교 - 구현 단계에서 확정)

[시각화]
생애주기 곡선 / 클러스터 분포도 (Plotly)
```
